# > 평범한 한글 v0.1
함수형 난해한 언어 '평범한 한글'의 명세와 구현체입니다.
'평범한 한글' 프로그램은 한글과 그 외 문자로 이루어진 문자열입니다.
여기서 한글은 다음의 범위에 들어가는 유니코드 문자를 말합니다.

* U+1100-U+1112 (한글 자모 중 초성)
* U+3131-U+314E (호환용 한글 자모 중 자음)
* U+AC00-U+D7AF (한글 음절)
* U+FFA1-U+FFBE (반각 한글 자모 중 자음)

한글 음절은 초성만 추출하여 한글 자모와 같이 취급하며, 한글 자모와 반각 한글 자모는 대응되는 호환용 한글 자모로 변환됩니다. 편의를 위해 언어 명세는 호환용 한글 자모로 기술했습니다. 한편 한글이 아닌 문자는 달리 구분하지 않고 공백과 같이 생각합니다. 따라서 이후의 설명에도 한글이 아닌 문자는 공백이 대표합니다.

## ㄱ. 명세

거센소리와 된소리는 예사소리와 같게 취급하며, 겹자음은 자음 두 개로 분리하여 생각합니다. 이는 작문 시 단어 선택의 폭을 넓히기 위한 선택입니다. 예시로, 다음 단어쌍들은 동일합니다.

* `ㄱㄲㅅㄹ` = `ㄱㄱㅅㄹ`
* `ㅇㅊㅂㅌ` = `ㅇㅈㅂㄷ`
* `ㄳㅎㄴㄷ` = `ㄱㅅㅎㄴㄷ`

간결한 설명을 위해 예사소리 단자음 10개 `ㄱㄴㄷㄹㅁㅂㅅㅇㅈㅎ`만 사용하여 설명하겠습니다. 또한, 모든 `ㅎ`은 그 앞에 공백이 있는 것과 없는 것을 동일하게 봅니다. 따라서 편의를 위해 모든 `ㅎ` 앞에 공백이 있는 것으로 생각합니다.

### 가. 정수 리터럴
`ㅇ`과 `ㅎ`을 제외한 8개 자음 `ㄱㄴㄷㄹㅁㅂㅅㅈ`은 8진수 가변 길이 정수를 부호화하기 위한 목적으로**만** 사용됩니다.

* 자음 `ㄱ, ㄴ, ㄷ, ㄹ, ㅁ, ㅂ, ㅅ, ㅈ`은 각각 8진수 숫자 0, 1, 2, 3, 4, 5, 6, 7을 뜻합니다.
* 가장 낮은 자리수가 가장 먼저 옵니다. 즉, 역순으로 부호화합니다. 예를 들어, 8진수 210은 `ㄱㄴㄷ`로 적습니다.
* 자음 개수가 홀수면 양수를, 자음 개수가 짝수면 음수를 부호화합니다. 뒤에 `ㄱ`(0)을 덧붙이는 방법으로 원하는 부호를 지정할 수 있습니다.

예시는 다음과 같습니다.

* `ㄱ` (oct +0) = 0
* `ㄴ` (oct +1) = 1
* `ㄴㄱ` (oct -01) = -1
* `ㄴㄱㄱ` (oct +001) = 1
* `ㄱㄴ` (oct -10) = -8
* `ㄱㄴㄱ` (oct +010) = +8
* `ㄱㄱㄴ` (oct +100) = +64
* `ㄱㄱㄴㄱ` (oct -0100) = -64

### 나. 단어
단어란 공백을 포함하지 않는 가장 긴 연속한 한글 자음열을 말합니다. 예를 들어 프로그램 `ㄱ ㄴㄷ   ㄱ ㅎㄷ`은 단어 네 개 `ㄱ`, `ㄴㄷ`, `ㄱ`, `ㅎㄷ`으로 이루어져 있습니다.

### 다. 객체와 타입
객체는 프로그램 실행의 기본 단위입니다. 현재 객체가 가질 수 있는 타입은 실수(Number)와 함수(Closure) 두 가지입니다. 실수 객체는 8개 자음 `ㄱㄴㄷㄹㅁㅂㅅㅈ`으로만 단어를 구성한 정수 리터럴로 생성하거나, 함수의 호출 결과로 생성될 수 있습니다. 함수는 함수 정의로 생성하거나, 마찬가지로 다른 함수의 호출 결과로 생성될 수 있습니다.

### 라. 함수 정의
모든 함수는 0개 이상의 객체를 인수로 받아 객체 하나를 돌려줍니다. 함수를 정의할 때 이 함수가 몇 개의 인수를 요구하는지는 따로 표기하지 않으며, 인수 배열의 형태로 접근합니다. 함수를 정의하는 구문은 다음과 같이 함수 몸통이 되는 객체 하나와 단어 `ㅎ`으로 이루어지며, 구문 전체가 함수 객체가 됩니다.

`[함수 몸통] ㅎ`

예를 들어 인수에 상관 없이 실수 3을 돌려주는 함수는 `ㄹ ㅎ`과 같이 정의합니다.

### 마. 함수 호출
인수 n개를 써서 함수를 호출하는 구문은 다음과 같이, (n+1)개의 객체에 이어 `ㅎ` 뒤에 인수 개수 n을 부호화한 정수 리터럴을 붙인 단어 1개로 구성됩니다. 마지막 객체가 호출할 함수이고, 첫 n개의 객체가 이 함수에 인수로 전달됩니다. 

`[인수0] [인수1] [인수2] ... [인수n-1] [함수] ㅎ[정수 리터럴 n]`

인수에 상관 없이 실수 3을 돌려주는 함수에 두 개의 인수 1과 -1을 전달하여 호출하는 구문은 다음과 같습니다.

`ㄴ ㄴㄱ ㄹ ㅎ ㅎㄷ`

#### 기본 제공 함수
본래 호출할 함수는 함수 객체여야 하지만, 정수 리터럴에 한해 실수 객체를 허용합니다. 이 때의 행동은 나열된 바와 같습니다. 참고로 불 대수 True와 False는 [Church 인코딩](https://en.wikipedia.org/wiki/Church_encoding#Church_Booleans)을 사용합니다. 즉, True를 λ(x,y).x로, False를 λ(x,y).y로 정의합니다.

* `ㄱ` 곱셈
  * 예시: `ㄱㄴ ㄷㄹ ㅁ ㄱ ㅎㄹ` = -0o10 * -0o32 * 0o4 = 0o1500 = 832
* `ㄷ` 덧셈
  * 예시: `ㄱㄴ ㄷㄹ ㅁ ㄷ ㅎㄹ` = -0o10 + -0o32 + 0o4 = -0o36 = -30
* `ㅅ` 거듭제곱
  * 예시: `ㄷ ㄹ ㅅ ㅎㄷ` = 2 ** 3 = 8
* `ㄴ` 같다
  * 예시: `ㄱ ㄱㄱ ㄴ ㅎㄷ` = (0 == -0) = True
* `ㅈ` 보다 작다
  * 예시: `ㄴㄱ ㄴ ㅈ ㅎㄷ` = (-1 < 1) = True

### 바. 인수 접근
함수를 정의할 때 함수 몸통에서 그 함수에 전달될 n번째 인수를 사용하려면 다음과 같이 `ㅇ` 앞에 인수 위치 n을 부호화한 정수 리터럴을 붙인 단어로 접근할 수 있습니다. 참고로 n이 음수이면 절댓값을 취해 양수로 변환합니다.

`[정수 리터럴 n]ㅇ`

예를 들어 0번째 인수를 반환하는 함수는 다음과 같습니다. (참고로 이 함수는 Church-encoded True와 같습니다.)

`ㄱㅇ ㅎ`

또, 0번째 인수와 1번째 함수를 더해 반환하는 함수는 다음과 같습니다.

`ㄱㅇ ㄴㅇ ㄷ ㅎㄷ ㅎ`

### 사. 상위 함수의 인수 접근
정의하려는 함수를 둘러싼 함수들의 인수도 접근할 수 있습니다. 정의하려는 함수를 0번째, 이를 바로 둘러싼 함수를 1번째, 다시 1번째 함수를 바로 둘러싼 함수를 2번째로 번호를 매깁니다. 이 번호는 함수(Closure) 객체 안에 환경(Environment)으로서 저장되어 함수 객체를 어느 문맥에서 호출하더라도 일관된 행동을 보장합니다. m번째 함수의 n번째 인수는 다음과 같이 접근합니다. 참고로 m이나 n이 음수이면 절댓값을 취해 양수로 변환합니다.

`[정수 리터럴 n]ㅇ[정수 리터럴 m]`

예를 들어 함수 λx.λy.(x+y)는 다음과 같이 작성할 수 있습니다.

`ㄱㅇㄴ ㄱㅇㄱ ㄷ ㅎㄷ ㅎ ㅎ`

이 함수에 차례로 3과 4를 인수로 넣어 호출하는 구문은 다음과 같습니다.

`ㄹ ㅁ ㄱㅇㄴ ㄱㅇㄱ ㄷ ㅎㄷ ㅎ ㅎ ㅎㄴ ㅎㄴ`

### 자. 재귀 함수
정의하려는 함수 및 그를 둘러싼 함수에 대한 접근도 가능합니다. 앞서와 같이 함수에 번호를 매겨, n번째 함수를 다음과 같이 접근할 수 있습니다.

`ㅇ[정수 리터럴 n]`

예를 들어 1번째 함수를 접근하려면 다음과 같이 합니다.

`ㅇㄴ`


## ㄴ. 예시

## ㄷ. 구현체 사용법
* `python pbhhg.py` => stdin에서 줄별로 문자열을 읽어 각각을 실행해 결과 출력
* `python pbhhg.py [string]` => 주어진 문자열을 실행해서 결과 출력
* 결과로 실수만 여러 개가 나올 경우 문자열로 변환해 출력하도록 했지만, 명세의 일부는 아닙니다.
